## 1. Tipos de datos

JavaScript tiene 7 tipos de datos primitivos. Antes de operar con ellos, hay que saber qué tipos existen.

### 1.1. Number

Para enteros, decimales y valores especiales como `Infinity` y `NaN`.

```js
let edad = 25;          // Entero
let precio = 9.99;      // Decimal

// 1.1.1. Constantes especiales
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(Infinity);                // Infinito
console.log(NaN);                     // "Not a Number"

// 1.1.2. Precisión y zonas de seguridad
console.log(0.1 + 0.2); // 0.30000000000000004 ⚠️ (cuidado con decimales)
console.log(Number.isSafeInteger(9007199254740991)); // true

// 1.1.3. Uso del guión bajo (_) para legibilidad
let millon = 1_000_000;
console.log(millon); // 1000000
```

---

### 1.2. String

Texto entre comillas simples, dobles, o Template Strings con backticks.

```js
let nombre = "Ana";
let saludo = 'Hola';

// 1.2.1. Caracteres especiales
let texto = "Ella dijo: \"Hola\"";  // Comillas dentro de string
let ruta = "C:\\Users\\Ana";         // Barra invertida
let linea = "Línea 1\nLínea 2";     // Salto de línea

// 1.2.2. Template String (comillas invertidas `)
let edad = 20;
let mensaje = `Mi nombre es ${nombre} y tengo ${edad} años.`;
console.log(mensaje); // "Mi nombre es Ana y tengo 20 años."
```

---

### 1.3. Boolean

```js
let esMayorDeEdad = true;
let estaLloviendo = false;

console.log(10 > 5);   // true
console.log(10 === 5); // false
```

---

### 1.4. Undefined

```js
let variableSinValor;
console.log(variableSinValor); // undefined
// Una variable declarada pero sin asignar = undefined
```

---

### 1.5. Null

```js
let respuesta = null;
// null significa "intencionalmente vacío"
console.log(respuesta); // null
```

> 💡 **Diferencia clave**: `undefined` = no se asignó valor. `null` = se asignó "nada" a propósito.

---

### 1.6. Symbol

```js
let id1 = Symbol("id");
let id2 = Symbol("id");
console.log(id1 === id2); // false — cada Symbol es único
```

---

### 1.7. BigInt

```js
let numeroGrande = 9007199254740991n; // La "n" al final lo hace BigInt
console.log(numeroGrande + 1n); // 9007199254740992n
```

## 2. Operadores

Los operadores permiten realizar acciones con los datos.

### 2.1. ¿Qué son los operadores?

Los operadores son símbolos especiales que nos permiten manipular datos, realizar cálculos y tomar decisiones en el código.

### 2.2. Aritméticos

```js
console.log(5 + 3);  // 8  — Suma
console.log(5 - 3);  // 2  — Resta
console.log(5 * 3);  // 15 — Multiplicación
console.log(10 / 2); // 5  — División
console.log(10 % 3); // 1  — Módulo (resto)
console.log(2 ** 3); // 8  — Potencia
```

### 2.3. Comparación

```js
console.log(5 == "5");  // true  — Compara valor (con conversión)
console.log(5 === "5"); // false — Compara valor Y tipo ✅ (usar siempre este)
console.log(5 !== 3);   // true
console.log(10 > 5);    // true
console.log(10 <= 10);  // true
```

### 2.4. Asignación

```js
let x = 10;
x += 5;  // x = x + 5 → 15
x -= 3;  // x = x - 3 → 12
x *= 2;  // x = x * 2 → 24
x /= 4;  // x = x / 4 → 6
x **= 2; // x = x ** 2 → 36
```

### 2.5. Concatenación de cadenas

```js
let nombre = "Carlos";
let apellido = "López";
let nombreCompleto = nombre + " " + apellido;
console.log(nombreCompleto); // "Carlos López"

// Con template string (más recomendado)
console.log(`Hola, ${nombre}!`); // "Hola, Carlos!"
```

## 3. Condicionales

Las estructuras condicionales en JavaScript permiten ejecutar código solo cuando se cumplen ciertas condiciones (una condición es cualquier expresión que, al evaluarse, resulta un booleano). Estas condiciones se basan en valores booleanos (true o false).

### 3.1. if, else if y else

```js
let temperatura = 35;

if (temperatura > 30) {
  console.log("Hace mucho calor 🥵");
} else if (temperatura > 20) {
  console.log("Está agradable 😊");
} else {
  console.log("Hace frío 🥶");
}
// Resultado: "Hace mucho calor 🥵"
```

### 3.2. Operador Ternario

Forma corta de un if/else en una sola línea.

```js
// condición ? si_es_true : si_es_false
let edad = 18;
let acceso = edad >= 18 ? "Puede entrar" : "No puede entrar";
console.log(acceso); // "Puede entrar"
```

### 3.3. console.assert()

```js
// Lanza error si la condición es falsa — útil para depurar
console.assert(2 + 2 === 4, "Las matemáticas fallaron");  // (sin error)
console.assert(2 + 2 === 5, "Las matemáticas fallaron");  // ❌ Error en consola
```

### 3.4. Estructura switch

Útil cuando hay varias condiciones de igualdad (`===`) sobre la misma variable.

```js
let dia = "lunes";

switch (dia) {
  case "lunes":
    console.log("Inicio de semana 💪");
    break;
  case "viernes":
    console.log("¡Por fin viernes! 🎉");
    break;
  default:
    console.log("Día normal");
}
```

### 3.5. Valores Truthy y Falsy

Valores que se comportan como `false` o `true` dentro de una condición.

```js
// FALSY (se comportan como false):
// false, 0, "", null, undefined, NaN

if (0)    { console.log("No se ejecuta"); }
if ("")   { console.log("No se ejecuta"); }
if (null) { console.log("No se ejecuta"); }

// TRUTHY (se comportan como true):
// Cualquier número distinto de 0, cualquier string no vacío, arrays, objetos

if (1)       { console.log("Sí se ejecuta ✅"); }
if ("hola")  { console.log("Sí se ejecuta ✅"); }
if ([])      { console.log("Array vacío es truthy ✅"); }
```

## 4. Bucles

Los bucles en JavaScript nos permiten repetir una serie de instrucciones varias veces sin tener que escribir el mismo código repetidamente.

### 4.1. for

```js
// for (inicialización; condición; incremento)
for (let i = 0; i < 5; i++) {
  console.log("Iteración número: " + i);
}
// Imprime del 0 al 4
```

- **4.1.1. Inicialización**: `let i = 0` — el contador empieza en 0.
- **4.1.2. Condición**: `i < 5` — el bucle se repite mientras sea `true`.
- **4.1.3. Incremento**: `i++` — suma 1 al contador en cada vuelta.

### 4.2. while

```js
let contador = 0;

while (contador < 3) {
  console.log("Contador: " + contador);
  contador++;
}
// Mientras la condición sea true, repite
```

### 4.3. do...while

```js
let numero = 0;

do {
  console.log("Se ejecuta al menos una vez: " + numero);
  numero++;
} while (numero < 3);
// Se ejecuta PRIMERO, luego verifica la condición
```

### 4.4. Interrupción y continuación en bucles

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) break;    // Para el bucle al llegar a 5
  if (i === 2) continue; // Salta el 2 y continúa
  console.log(i);
}
// Imprime: 0, 1, 3, 4
```

## 5. Funciones

Las funciones permiten agrupar código reutilizable. Requiere entender tipos, operadores y condicionales primero.

### 5.1. Funciones declaradas

```js
function saludar(nombre) {
  return "Hola, " + nombre + "!";
}

console.log(saludar("Ana")); // "Hola, Ana!"
```

### 5.2. Funciones Expresadas o Anónimas

```js
const sumar = function(a, b) {
  return a + b;
};

console.log(sumar(3, 4)); // 7
```

### 5.3. Funciones Autoejecubles

```js
(function() {
  console.log("Me ejecuto sola al cargar!");
})();
// Se ejecuta inmediatamente, sin necesidad de llamarla
```

### 5.4. Funciones Flecha

```js
// Forma corta y moderna
const multiplicar = (a, b) => a * b;
console.log(multiplicar(5, 3)); // 15

// Con cuerpo de función
const saludar = (nombre) => {
  let mensaje = `Hola, ${nombre}!`;
  return mensaje;
};
```

### 5.5. Detalles adicionales de las funciones

```js
// Parámetros por defecto
function bienvenida(nombre = "visitante") {
  return `Bienvenido, ${nombre}`;
}
console.log(bienvenida());        // "Bienvenido, visitante"
console.log(bienvenida("Pedro")); // "Bienvenido, Pedro"
```

## 6. Estructuras de datos

En JavaScript, las estructuras de datos son fundamentales para organizar y almacenar la información. Las dos principales son los **arrays** y los **objetos**. Ambos permiten trabajar con colecciones de datos, pero cada uno tiene características específicas que los hacen adecuados para distintos tipos de tareas.

### 6.1. Array

```js
let frutas = ["manzana", "banana", "cereza"];
```

#### 6.1.1. Acceder a los elementos

```js
console.log(frutas[0]); // "manzana" (índice empieza en 0)
console.log(frutas[2]); // "cereza"
```

#### 6.1.2. Modificar un elemento

```js
frutas[1] = "mango";
console.log(frutas); // ["manzana", "mango", "cereza"]
```

#### 6.1.3. Acceder a un índice que no existe

```js
console.log(frutas[10]); // undefined
```

#### 6.1.4. Modificar un índice fuera del rango

```js
frutas[5] = "uva";
console.log(frutas); // ["manzana", "mango", "cereza", empty, empty, "uva"]
```

#### 6.1.5. Recorrer una lista

```js
frutas.forEach(function(fruta) {
  console.log(fruta);
});
```

---

### 6.2. Objeto

```js
let persona = {
  nombre: "Laura",
  edad: 25,
  ciudad: "Bogotá"
};
```

#### 6.2.1. Acceder a una propiedad

```js
console.log(persona.nombre);  // "Laura"
console.log(persona["edad"]); // 25
```

#### 6.2.2. Modificar una propiedad

```js
persona.edad = 26;
console.log(persona.edad); // 26
```

#### 6.2.3. Acceder a una propiedad que no existe

```js
console.log(persona.telefono); // undefined
```

#### 6.2.4. Agregar una nueva propiedad

```js
persona.email = "laura@email.com";
console.log(persona);
// { nombre: "Laura", edad: 26, ciudad: "Bogotá", email: "laura@email.com" }
```

## 7. Temas adicionales

Conceptos que complementan todo lo anterior una vez tienes la base sólida.

### 7.1. Estructura condicional switch (avanzado)

Cuando hay múltiples casos, se pueden agrupar:

```js
let mes = 2;

switch (mes) {
  case 1:
  case 3:
  case 5:
    console.log("Mes con 31 días");
    break;
  case 2:
    console.log("Mes con 28 o 29 días");
    break;
  default:
    console.log("Otro mes");
}
```

### 7.2. Valores Truthy y Falsy (profundización)

```js
// Uso práctico: valor por defecto con ||
let nombre = "" || "Anónimo";
console.log(nombre); // "Anónimo" — porque "" es falsy

// Uso práctico: ejecutar solo si existe con &&
let usuario = { nombre: "Ana" };
usuario && console.log(usuario.nombre); // "Ana"
```

### 7.3. Uso de `const` con listas u objetos

`const` no significa inmutable para arrays y objetos. Solo significa que no puedes reasignar la variable.

```js
const numeros = [1, 2, 3];
numeros.push(4);       // ✅ Esto sí funciona
console.log(numeros);  // [1, 2, 3, 4]

// numeros = [5, 6, 7]; // ❌ Error — no puedes reasignar

const auto = { marca: "Toyota" };
auto.color = "rojo";   // ✅ Puedes agregar propiedades
console.log(auto);     // { marca: "Toyota", color: "rojo" }
```
