# Semana 3 — DOM, Eventos y Persistencia de Datos en JavaScript

---

# 1. Document Object Model (DOM)

## 1.1 ¿Qué es el DOM y cuál es su relación con HTML y JS?

HTML define la **estructura** de una página web (qué se va a mostrar), mientras que JavaScript controla el **comportamiento** (cómo y cuándo se muestran o cambian esos elementos). El puente entre ambos es el **DOM**.

> **DOM (Document Object Model):** Es una representación estructurada en forma de árbol del contenido de un documento web. Actúa como interfaz entre el HTML y JavaScript, permitiendo que los scripts accedan y modifiquen los elementos de la página de forma dinámica.

**Ejemplo — JS accediendo al DOM para cambiar texto:**

```html
<!DOCTYPE html>
<html>
  <body>
    <h1 id="titulo">Hola mundo</h1>
    <script>
      // JS accede al DOM y cambia el texto del h1
      document.getElementById("titulo").textContent = "¡DOM en acción!";
    </script>
  </body>
</html>
```

> Sin el DOM, JavaScript no tendría forma de "ver" ni modificar el HTML.

---

## 1.2 Características del DOM

### Estructura en árbol
El DOM organiza el contenido del documento en una jerarquía de nodos. Cada etiqueta HTML se convierte en un nodo, y los nodos se relacionan entre sí como **padre**, **hijo** o **hermano**.

### Accesibilidad dinámica
A través de métodos del DOM, es posible **agregar, modificar o eliminar** contenido en tiempo real sin recargar la página.

### Independencia del lenguaje
Aunque en el navegador se usa principalmente con JavaScript, el DOM es un estándar independiente del lenguaje. Puede ser manipulado desde otros lenguajes como Python (en entornos de automatización) o C#.

---

## 1.3 ¿Cómo funciona el DOM?

Cuando el navegador carga una página, toma el código HTML y lo convierte en un conjunto de **objetos (nodos)** que forman el árbol DOM. Esto permite que JavaScript manipule esos nodos en tiempo real.

**Ejemplo — cambios sin recargar la página:**

```js
// Cambiar el color de fondo
document.body.style.backgroundColor = "lightblue";

// Cambiar el texto de un párrafo
document.querySelector("p").textContent = "Texto actualizado dinámicamente.";

// Agregar un nuevo elemento
const nuevoParrafo = document.createElement("p");
nuevoParrafo.textContent = "Soy un párrafo nuevo.";
document.body.appendChild(nuevoParrafo);
```

Con estas tres líneas se puede: cambiar estilos, modificar contenido y agregar elementos, todo sin recargar la página.

---

## 1.4 El Árbol del DOM

Cada etiqueta HTML se convierte en un **nodo** dentro del árbol, y los elementos se organizan reflejando relaciones **padre-hijo**.

**HTML de ejemplo:**

```html
<html>
  <head>
    <title>Ejemplo</title>
  </head>
  <body>
    <h1>Título</h1>
    <p>Párrafo</p>
  </body>
</html>
```

**Representación del árbol:**

```
Document
└── html
    ├── head
    │   └── title
    │       └── "Ejemplo"  ← nodo de texto
    └── body
        ├── h1
        │   └── "Título"   ← nodo de texto
        └── p
            └── "Párrafo"  ← nodo de texto
```

---

## 1.5 Tipos de Nodos en el DOM

| Tipo de nodo | Descripción | Ejemplos |
|---|---|---|
| **Nodo de elemento** | Representa una etiqueta HTML | `<div>`, `<p>`, `<h1>` |
| **Nodo de texto** | El contenido textual dentro de un elemento | `"Hola mundo"` |
| **Nodo de atributo** | Atributos de los elementos HTML | `id`, `class`, `src` |
| **Nodo de comentario** | Comentarios en el código HTML | `<!-- comentario -->` |

**Ejemplo — identificar tipos de nodo con JS:**

```js
const parrafo = document.querySelector("p");

console.log(parrafo.nodeType);      // 1 → Nodo de elemento
console.log(parrafo.firstChild.nodeType); // 3 → Nodo de texto
```

> `nodeType === 1` es un elemento HTML. `nodeType === 3` es un nodo de texto.

# 2. Manipulación del DOM y Persistencia de Datos

## 2.1 Selección de Elementos del DOM

Antes de manipular un elemento, hay que **seleccionarlo**. JavaScript ofrece varios métodos para esto:

### `document.getElementById(id)`
Selecciona **un único elemento** por su atributo `id`. Retorna el elemento o `null` si no existe.

```js
const titulo = document.getElementById("titulo");
console.log(titulo); // <h1 id="titulo">...</h1>
```

### `document.getElementsByClassName(className)`
Selecciona **todos los elementos** que tienen la clase especificada. Devuelve una `HTMLCollection` (parecida a un array, pero no es un array).

```js
const tarjetas = document.getElementsByClassName("tarjeta");
console.log(tarjetas.length); // número de elementos con clase "tarjeta"
```

### `document.getElementsByTagName(tagName)`
Selecciona **todos los elementos** que tienen una etiqueta HTML específica. Retorna una `HTMLCollection`.

```js
const parrafos = document.getElementsByTagName("p");
console.log(parrafos[0]); // primer <p> del documento
```

### `document.querySelector(selector)`
Selecciona **el primer elemento** que coincide con un selector CSS. Devuelve un solo elemento o `null`.

```js
const boton = document.querySelector(".btn-primario");
// También acepta selectores complejos:
const input = document.querySelector("form input[type='email']");
```

### `document.querySelectorAll(selector)`
Selecciona **todos los elementos** que coinciden con un selector CSS. Retorna una `NodeList` (se puede iterar con `forEach`).

```js
const items = document.querySelectorAll("ul li");
items.forEach(item => console.log(item.textContent));
```

> **¿Cuál usar?** En la mayoría de casos modernos se prefiere `querySelector` y `querySelectorAll` por su flexibilidad con selectores CSS.

---

## 2.2 Modificación del DOM

Una vez seleccionado un elemento, se puede modificar su **contenido**, **atributos** o **estilos**.

### 2.2.1 Modificación de Contenido

| Propiedad | Descripción | ¿Interpreta HTML? |
|---|---|---|
| `textContent` | Cambia o recupera el texto plano del elemento | ❌ No |
| `innerHTML` | Obtiene o cambia el HTML interno del elemento | ✅ Sí |
| `outerHTML` | Como `innerHTML`, pero incluye la etiqueta del propio elemento | ✅ Sí |

```js
const div = document.querySelector("#caja");

// textContent: solo texto, no interpreta HTML
div.textContent = "<strong>Hola</strong>"; // Muestra el texto literal

// innerHTML: interpreta las etiquetas HTML
div.innerHTML = "<strong>Hola</strong>"; // Muestra "Hola" en negrita

// outerHTML: reemplaza el elemento completo
console.log(div.outerHTML); // "<div id='caja'><strong>Hola</strong></div>"
```

> ⚠️ Usar `innerHTML` con datos del usuario puede ser un riesgo de seguridad (XSS). Preferir `textContent` cuando no se necesita HTML.

---

### 2.2.2 Modificación de Atributos

```js
const imagen = document.querySelector("img");

// Agrega o modifica el atributo src
imagen.setAttribute("src", "foto-nueva.jpg");

// Recupera el valor del atributo alt
const descripcion = imagen.getAttribute("alt");
console.log(descripcion); // "descripción de la imagen"

// Elimina el atributo alt
imagen.removeAttribute("alt");
```

---

### 2.2.3 Modificación de Estilos con `.style`

La propiedad `style` permite modificar **estilos en línea** directamente desde JavaScript. Las propiedades CSS que tienen guión se escriben en **camelCase**.

```js
const caja = document.querySelector(".caja");

caja.style.backgroundColor = "tomato";  // background-color → backgroundColor
caja.style.fontSize = "24px";           // font-size → fontSize
caja.style.display = "none";            // ocultar elemento
```

> ⚠️ `.style` solo aplica estilos **en línea**. Para estilos más mantenibles, se prefiere usar `classList`.

---

### 2.2.4 Modificación de Clases con `classList`

`classList` es la forma más limpia y eficiente de manejar las clases de un elemento.

```js
const boton = document.querySelector("#miBoton");

boton.classList.add("activo");         // Agrega la clase "activo"
boton.classList.remove("activo");      // Elimina la clase "activo"
boton.classList.toggle("activo");      // Si existe → la quita; si no → la agrega
boton.classList.contains("activo");    // Retorna true/false
```

**Ejemplo práctico — botón que cambia de estado:**

```js
const boton = document.querySelector("#btn");

boton.addEventListener("click", () => {
  boton.classList.toggle("activo");
  // En el CSS: .activo { background: green; color: white; }
});
```

---

## 2.3 Creación, Clonación y Eliminación de Elementos

### Crear e insertar elementos

```js
// 1. Crear el elemento
const nuevoItem = document.createElement("li");

// 2. Darle contenido
nuevoItem.textContent = "Nuevo elemento de lista";

// 3. Insertarlo en el DOM
const lista = document.querySelector("ul");
lista.appendChild(nuevoItem);  // Lo agrega al final

// insertBefore(nuevo, referencia) → lo agrega antes de un elemento específico
const referencia = lista.children[0];
lista.insertBefore(nuevoItem, referencia); // Lo agrega antes del primer li
```

---

### Clonar elementos

```js
const tarjeta = document.querySelector(".tarjeta");

// cloneNode(true) → clona el elemento Y todo su contenido (hijos incluidos)
// cloneNode(false) → clona solo el elemento vacío
const copia = tarjeta.cloneNode(true);

document.querySelector(".contenedor").appendChild(copia);
```

---

### Reemplazar elementos

```js
const nuevoTitulo = document.createElement("h2");
nuevoTitulo.textContent = "Título actualizado";

const tituloViejo = document.querySelector("h1");
tituloViejo.parentNode.replaceChild(nuevoTitulo, tituloViejo);
// El h1 es reemplazado por el nuevo h2
```

---

### Eliminar elementos

```js
const elemento = document.querySelector(".eliminar");
elemento.remove(); // Elimina el elemento directamente del DOM
```

---

## 2.4 Persistencia de Datos: `localStorage` y `sessionStorage`

Ambas APIs permiten guardar datos en el navegador del usuario en pares **clave-valor**, pero difieren en su **duración**.

### 2.4.1 `localStorage`

Los datos guardados en `localStorage` **persisten entre sesiones**: siguen disponibles aunque el usuario cierre el navegador y lo vuelva a abrir, hasta que se eliminen manualmente.

**Características:**
- Almacena pares clave-valor (solo strings).
- Límite aproximado de **5 MB**.
- Accesible desde cualquier página del mismo dominio.
- Persiste aunque se cierre el navegador.

```js
// Guardar
localStorage.setItem("usuario", "Daniel");
localStorage.setItem("tema", "oscuro");

// Recuperar
const usuario = localStorage.getItem("usuario");
console.log(usuario); // "Daniel"

// Eliminar un dato específico
localStorage.removeItem("tema");

// Eliminar todo
localStorage.clear();
```

**Guardar objetos (se debe usar JSON):**

```js
const config = { idioma: "es", notificaciones: true };

// Guardar objeto → convertir a string con JSON.stringify
localStorage.setItem("config", JSON.stringify(config));

// Recuperar objeto → convertir de string con JSON.parse
const configGuardada = JSON.parse(localStorage.getItem("config"));
console.log(configGuardada.idioma); // "es"
```

---

### 2.4.2 `sessionStorage`

Funciona igual que `localStorage`, pero los datos **solo duran mientras la pestaña está abierta**. Al cerrar la pestaña o el navegador, los datos se eliminan automáticamente.

**Características:**
- Mismos métodos que `localStorage`.
- Límite aproximado de **5 MB**.
- Los datos son **exclusivos por pestaña** — no se comparten entre pestañas distintas.
- Se elimina al cerrar la pestaña.

```js
// Guardar temporalmente (por ejemplo, un paso de un formulario de varios pasos)
sessionStorage.setItem("paso_actual", "2");

// Recuperar
const paso = sessionStorage.getItem("paso_actual");
console.log(paso); // "2"

// Eliminar
sessionStorage.removeItem("paso_actual");

// Eliminar todo
sessionStorage.clear();
```

**Comparación rápida:**

| Característica | `localStorage` | `sessionStorage` |
|---|---|---|
| Duración | Permanente | Solo mientras la pestaña esté abierta |
| Compartido entre pestañas | ✅ Sí | ❌ No |
| Límite | ~5 MB | ~5 MB |
| Se borra con `clear()` | ✅ Sí | ✅ Sí |

# 3. Recorrido del DOM y Eventos

## 3.1 Recorrido del DOM

El árbol del DOM se puede **navegar** moviéndose entre nodos relacionados: padres, hijos y hermanos. Esto es clave para manipular elementos de forma eficiente.

### Acceder al nodo padre — `parentNode`

```js
const parrafo = document.querySelector("p");
console.log(parrafo.parentNode); // El elemento que contiene al <p>, por ejemplo un <div>
```

---

### Acceder a los nodos hijos — `childNodes` y `children`

```js
const lista = document.querySelector("ul");

// childNodes: devuelve TODOS los nodos hijos (incluye nodos de texto y comentarios)
console.log(lista.childNodes); // NodeList [texto, li, texto, li, ...]

// children: devuelve SOLO los elementos HTML hijos (sin nodos de texto)
console.log(lista.children); // HTMLCollection [li, li, li]
console.log(lista.children[0]); // Primer <li>
```

---

### Acceder al primer y último hijo

```js
const lista = document.querySelector("ul");

// Primer hijo (puede ser un nodo de texto como un salto de línea)
console.log(lista.firstChild);

// Primer elemento HTML hijo (ignora nodos de texto)
console.log(lista.firstElementChild); // Primer <li>

// Último hijo
console.log(lista.lastChild);
console.log(lista.lastElementChild); // Último <li>
```

---

### Acceder a los hermanos — `nextSibling` y `previousSibling`

```js
const elemento = document.querySelector("#segundo");

// Hermano siguiente (puede incluir nodos de texto)
console.log(elemento.nextSibling);

// Siguiente hermano que sea elemento HTML
console.log(elemento.nextElementSibling);

// Hermano anterior
console.log(elemento.previousSibling);
console.log(elemento.previousElementSibling);
```

> En la mayoría de los casos se prefiere usar las variantes `...ElementSibling` y `...ElementChild` porque evitan los nodos de texto invisibles que HTML genera entre etiquetas.

---

## 3.2 ¿Qué son los Eventos en el DOM?

Un **evento** es cualquier interacción que ocurre en la página web a la cual el navegador puede responder: un clic, una tecla presionada, el envío de un formulario, el scroll de la página, etc.

Cuando ocurre un evento, se puede activar una función JavaScript específica llamada **event handler** (manejador de eventos). Esto es lo que hace posible que una página web sea interactiva.

```js
// Ejemplo básico: responder a un clic
const boton = document.querySelector("#miBoton");

boton.addEventListener("click", function() {
  alert("¡Hiciste clic en el botón!");
});
```

---

## 3.3 Tipos de Eventos Comunes

### 3.3.1 Eventos de Ratón

| Evento | Cuándo se dispara |
|---|---|
| `click` | Al hacer clic en un elemento |
| `dblclick` | Al hacer doble clic |
| `mouseover` | Cuando el cursor entra al elemento |
| `mouseout` | Cuando el cursor sale del elemento |
| `mousemove` | Cada vez que el ratón se mueve dentro del elemento |

```js
const caja = document.querySelector(".caja");

caja.addEventListener("mouseover", () => {
  caja.style.backgroundColor = "lightyellow";
});

caja.addEventListener("mouseout", () => {
  caja.style.backgroundColor = "";
});
```

---

### 3.3.2 Eventos de Teclado

| Evento | Cuándo se dispara |
|---|---|
| `keydown` | Al presionar una tecla (se dispara continuamente si se mantiene) |
| `keyup` | Al soltar una tecla |
| `keypress` | *(Obsoleto)* Al presionar una tecla que produce un carácter |

```js
document.addEventListener("keydown", (e) => {
  console.log("Tecla presionada:", e.key); // "a", "Enter", "ArrowUp", etc.
});
```

> `keypress` está obsoleto. Se recomienda usar `keydown` o `keyup`.

---

### 3.3.3 Eventos de Formulario

| Evento | Cuándo se dispara |
|---|---|
| `submit` | Al enviar el formulario |
| `change` | Cuando el valor de un campo cambia y pierde el foco |
| `input` | Cada vez que se escribe algo en un campo de texto |

```js
const formulario = document.querySelector("form");
const inputNombre = document.querySelector("#nombre");

// Mostrar el valor mientras el usuario escribe
inputNombre.addEventListener("input", (e) => {
  console.log("Escribiendo:", e.target.value);
});

// Interceptar el envío del formulario
formulario.addEventListener("submit", (e) => {
  e.preventDefault(); // Evita que la página se recargue
  console.log("Formulario enviado con:", inputNombre.value);
});
```

---

### 3.3.4 Eventos de Ventana

| Evento | Cuándo se dispara |
|---|---|
| `load` | Cuando la página y todos sus recursos (imágenes, CSS) han cargado |
| `resize` | Cuando la ventana cambia de tamaño |
| `scroll` | Cuando el usuario hace scroll en la página |

```js
window.addEventListener("resize", () => {
  console.log("Ancho actual:", window.innerWidth);
});

window.addEventListener("scroll", () => {
  console.log("Posición scroll:", window.scrollY);
});
```

---

### 3.3.5 Evento de Documento — `DOMContentLoaded`

Se dispara cuando el HTML ha sido completamente cargado y procesado, **sin esperar** a que terminen de cargar imágenes, hojas de estilo u otros recursos externos. Es más rápido que `load`.

```js
document.addEventListener("DOMContentLoaded", () => {
  console.log("El DOM está listo. Puedo manipular elementos con seguridad.");
  const titulo = document.querySelector("h1");
  titulo.textContent = "¡Página lista!";
});
```

---

## 3.4 Formas de Asignar Eventos

Existen tres maneras de escuchar eventos en JavaScript:

### 3.4.1 Asignación directa en HTML ❌ (no recomendada)

```html
<!-- Mezcla HTML con JS → mala práctica -->
<button onclick="alert('Clic!')">Haz clic</button>
```

> Dificulta el mantenimiento porque mezcla estructura (HTML) con comportamiento (JS).

---

### 3.4.2 Asignación con propiedades de eventos en JS ⚠️ (funciona pero limitada)

```js
const boton = document.querySelector("#btn");

boton.onclick = function() {
  console.log("Clic detectado");
};

// Problema: si asignas otra función, sobreescribe la anterior
boton.onclick = function() {
  console.log("Esto reemplaza el handler anterior");
};
```

---

### 3.4.3 `addEventListener()` ✅ (forma recomendada)

Permite agregar **múltiples eventos** al mismo elemento sin sobreescribir los anteriores.

```js
const boton = document.querySelector("#btn");

boton.addEventListener("click", () => {
  console.log("Primer handler");
});

boton.addEventListener("click", () => {
  console.log("Segundo handler — ambos se ejecutan");
});

// Para remover un listener, la función debe estar nombrada
function handleClick() {
  console.log("Este sí se puede remover");
}

boton.addEventListener("click", handleClick);
boton.removeEventListener("click", handleClick); // Lo remueve correctamente
```

## 3.5 Propagación de Eventos

Cuando ocurre un evento en un elemento, este se **propaga** a través del árbol del DOM en dos fases:

```
     Capturing (↓)            Bubbling (↑)
Document                 Document
  └── html                  └── html
       └── body                   └── body
            └── div                    └── div ← evento ocurre aquí
                 └── button ← clic     ...    y sube hacia arriba
```

### 3.5.1 Event Bubbling — Propagación hacia arriba ↑

El evento se dispara primero en el elemento que lo originó y luego **sube** hacia sus ancestros.

```js
document.querySelector("div").addEventListener("click", () => {
  console.log("Clic en el div");
});

document.querySelector("button").addEventListener("click", () => {
  console.log("Clic en el botón"); // Se ejecuta primero
});

// Si haces clic en el botón, verás:
// "Clic en el botón"
// "Clic en el div"  ← el evento subió (bubbling)
```

---

### 3.5.2 Event Capturing — Propagación hacia abajo ↓

El evento se captura desde el nodo raíz hacia el elemento que lo provocó. Para habilitarlo, se pasa `true` como tercer argumento de `addEventListener`.

```js
document.querySelector("div").addEventListener("click", () => {
  console.log("Captura en div");
}, true); // true activa el modo capturing

document.querySelector("button").addEventListener("click", () => {
  console.log("Captura en botón");
}, true);

// Si haces clic en el botón:
// "Captura en div"     ← va hacia abajo primero
// "Captura en botón"
```

---

### 3.5.3 Detener la propagación — `stopPropagation()`

```js
document.querySelector("button").addEventListener("click", (e) => {
  e.stopPropagation(); // El evento no sube más
  console.log("Clic en botón — no se propaga al div");
});

document.querySelector("div").addEventListener("click", () => {
  console.log("Esto NO se ejecutará si stopPropagation() fue llamado");
});
```

---

### 3.5.4 `event.target` vs `event.currentTarget`

| Propiedad | Retorna |
|---|---|
| `event.target` | El elemento **donde ocurrió originalmente** el evento (el que fue clickeado) |
| `event.currentTarget` | El elemento **al que está asignado** el listener en ese momento |

```js
document.querySelector("ul").addEventListener("click", (e) => {
  console.log("target:", e.target);          // El <li> específico que fue clickeado
  console.log("currentTarget:", e.currentTarget); // El <ul> donde está el listener
});
```

---

### 3.5.5 Delegación de Eventos

En lugar de asignar un listener a cada elemento hijo, se asigna **uno solo al elemento padre** y se aprovecha el bubbling para detectar en qué hijo ocurrió el evento. Muy útil cuando los hijos se crean dinámicamente.

```js
// ❌ Sin delegación: un listener por cada elemento
document.querySelectorAll("li").forEach(li => {
  li.addEventListener("click", () => console.log(li.textContent));
});

// ✅ Con delegación: un solo listener en el padre
document.querySelector("ul").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log("Clic en:", e.target.textContent);
  }
});
```

> La delegación es más eficiente en memoria y funciona automáticamente con elementos añadidos dinámicamente al DOM.

---

### 3.5.6 Prevenir el Comportamiento por Defecto — `preventDefault()`

Algunos elementos HTML tienen comportamientos por defecto (un link navega a otra página, un formulario recarga la página). `preventDefault()` los cancela.

```js
// Evitar que el formulario recargue la página
document.querySelector("form").addEventListener("submit", (e) => {
  e.preventDefault();
  console.log("Formulario interceptado — procesando con JS");
});

// Evitar que un link navegue
document.querySelector("a").addEventListener("click", (e) => {
  e.preventDefault();
  console.log("Link bloqueado");
});
```

---

### 3.5.7 Eventos Personalizados — `CustomEvent`

Se pueden crear y disparar eventos propios con `CustomEvent`. Útil para comunicación entre componentes.

```js
// Crear y disparar un evento personalizado
const miEvento = new CustomEvent("usuarioLogueado", {
  detail: { nombre: "Daniel", rol: "admin" } // datos adjuntos al evento
});

document.dispatchEvent(miEvento); // Lanzar el evento

// Escuchar el evento personalizado desde otro script
document.addEventListener("usuarioLogueado", (e) => {
  console.log("Usuario:", e.detail.nombre); // "Daniel"
  console.log("Rol:", e.detail.rol);        // "admin"
});
```
