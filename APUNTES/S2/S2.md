## 1. Estructuras de Datos en JavaScript: Objetos

### 1.1 Uso del punto y coma (`;`)

El punto y coma (`;`) en JavaScript es una buena práctica para marcar explícitamente el final de cada instrucción. Aunque JS tiene un mecanismo llamado **ASI** (Automatic Semicolon Insertion) que intenta insertarlos automáticamente, este puede producir comportamientos inesperados en código complejo o minificado.

```js
// ✅ Buena práctica
console.log("Texto1");
console.log("Texto2");

// ⚠️ Sin punto y coma — puede fallar en ciertos contextos
const a = 1
const b = 2
[a, b].forEach(console.log) // JS interpreta esto como: const b = 2[a, b].forEach(...)
```

### 1.2 Métodos en JavaScript

Un **método** es una función que está asociada a un objeto y que realiza acciones, generalmente sobre las propiedades de ese objeto.

```js
const persona = {
  nombre: "Ana",
  saludar() {               // método del objeto
    console.log("Hola, soy " + this.nombre);
  }
};

persona.saludar(); // "Hola, soy Ana"
```

### 1.3 Tipos de datos primitivos y sus métodos

Los tipos primitivos (`number`, `string`, `boolean`, etc.) no son objetos, pero JavaScript los **envuelve temporalmente** en objetos cuando se llama a un método sobre ellos. Esto se llama **boxing automático**.

#### Métodos de `Number`

| Método | Descripción |
|---|---|
| `toFixed(n)` | Redondea a `n` decimales y devuelve un string |
| `toString()` | Convierte el número en cadena de texto |
| `isNaN(valor)` | Verifica si el valor no es un número |
| `parseInt(str)` | Convierte un string a entero |
| `parseFloat(str)` | Convierte un string a decimal |

```js
const precio = 19.9876;

console.log(precio.toFixed(2));       // "19.99"
console.log(precio.toString());       // "19.9876"
console.log(Number.isNaN(NaN));       // true
console.log(Number.parseInt("42px")); // 42
console.log(Number.parseFloat("3.14abc")); // 3.14
```

#### Métodos de `String`

| Método | Descripción |
|---|---|
| `toUpperCase()` | Convierte a mayúsculas |
| `toLowerCase()` | Convierte a minúsculas |
| `trim()` | Elimina espacios al inicio y al final |
| `includes(sub)` | Verifica si contiene una subcadena |
| `split(sep)` | Divide el string en un array |
| `replace(a, b)` | Reemplaza una ocurrencia por otra |
| `slice(ini, fin)` | Extrae una porción del string |
| `length` | Propiedad: devuelve la longitud del string |

```js
const texto = "  Hola, Mundo!  ";

console.log(texto.trim());              // "Hola, Mundo!"
console.log(texto.trim().toLowerCase()); // "hola, mundo!"
console.log(texto.includes("Mundo"));   // true
console.log("a,b,c".split(","));        // ["a", "b", "c"]
console.log("Hola".slice(0, 2));        // "Ho"
console.log("Hola".replace("o", "0")); // "H0la"
```

### 1.4 Estructuras de datos en JavaScript

#### Arrays

Los arrays son **listas ordenadas de elementos** de cualquier tipo. Se acceden por índice (comenzando en `0`).

| Método            | Descripción                                                        |
| ----------------- | ------------------------------------------------------------------ |
| `push(elem)`      | Añade un elemento al final                                         |
| `pop()`           | Elimina y devuelve el último elemento                              |
| `shift()`         | Elimina y devuelve el primer elemento                              |
| `unshift(elem)`   | Añade un elemento al inicio                                        |
| `map(fn)`         | Aplica una función a cada elemento y devuelve un nuevo array       |
| `filter(fn)`      | Devuelve un nuevo array con los elementos que cumplen la condición |
| `find(fn)`        | Devuelve el primer elemento que cumple la condición                |
| `reduce(fn, ini)` | Acumula un valor a partir de todos los elementos                   |
| `includes(elem)`  | Verifica si el elemento está en el array                           |
| `length`          | Propiedad: número de elementos                                     |

```js
const nums = [1, 2, 3, 4, 5];

nums.push(6);                        // [1, 2, 3, 4, 5, 6]
nums.pop();                          // [1, 2, 3, 4, 5]

const dobles = nums.map(n => n * 2); // [2, 4, 6, 8, 10]
const pares  = nums.filter(n => n % 2 === 0); // [2, 4]
const suma   = nums.reduce((acc, n) => acc + n, 0); // 15

console.log(dobles); // [2, 4, 6, 8, 10]
console.log(pares);  // [2, 4]
console.log(suma);   // 15
```

#### Objetos

Un objeto es una **colección de pares clave-valor**. Las claves son strings (o Symbols) y los valores pueden ser de cualquier tipo.

| Método | Descripción |
|---|---|
| `Object.keys(obj)` | Devuelve un array con las claves del objeto |
| `Object.values(obj)` | Devuelve un array con los valores del objeto |
| `Object.entries(obj)` | Devuelve un array de pares `[clave, valor]` |
| `Object.assign(dest, src)` | Copia propiedades de un objeto a otro |

```js
const persona = { nombre: "Carlos", edad: 28, ciudad: "Medellín" };

console.log(Object.keys(persona));    // ["nombre", "edad", "ciudad"]
console.log(Object.values(persona));  // ["Carlos", 28, "Medellín"]
console.log(Object.entries(persona)); // [["nombre","Carlos"], ["edad",28], ["ciudad","Medellín"]]

// Recorrer un objeto con for...in
for (const clave in persona) {
  console.log(`${clave}: ${persona[clave]}`);
}
```

#### Sets

Un `Set` es una colección de **valores únicos**. No permite duplicados. Es útil para eliminar repetidos de un array.

| Método/Propiedad | Descripción |
|---|---|
| `add(valor)` | Añade un valor al Set |
| `has(valor)` | Verifica si el Set contiene el valor |
| `delete(valor)` | Elimina un valor del Set |
| `size` | Número de elementos en el Set |
| `clear()` | Elimina todos los elementos |

> ⚠️ **Corrección:** En los apuntes originales aparece `delate(valor)` — el método correcto es `delete(valor)`.

```js
const colores = new Set(["rojo", "verde", "rojo", "azul"]);
// Los duplicados se ignoran
console.log(colores); // Set { "rojo", "verde", "azul" }
console.log(colores.size); // 3

colores.add("amarillo");
colores.delete("verde");
console.log(colores.has("rojo")); // true

// Convertir Set a Array (para eliminar duplicados de un array)
const sinDuplicados = [...new Set([1, 2, 2, 3, 3, 4])]; // [1, 2, 3, 4]
```

---

#### Maps

Un `Map` es una colección de pares **clave-valor** donde tanto la clave como el valor pueden ser **de cualquier tipo** (incluyendo objetos, funciones, etc.), a diferencia de los objetos comunes donde las claves siempre son strings.

| Método/Propiedad | Descripción |
|---|---|
| `set(clave, valor)` | Añade o actualiza un par clave-valor |
| `get(clave)` | Devuelve el valor asociado a la clave |
| `has(clave)` | Verifica si existe la clave |
| `delete(clave)` | Elimina un par clave-valor |
| `size` | Número de pares en el Map |

> ⚠️ **Corrección:** En los apuntes originales aparece `self(clave, valor)` — el método correcto es `set(clave, valor)`.

```js
const inventario = new Map();

inventario.set("manzanas", 50);
inventario.set("peras", 30);
inventario.set("uvas", 20);

console.log(inventario.get("manzanas")); // 50
console.log(inventario.has("uvas"));     // true
console.log(inventario.size);            // 3

inventario.delete("peras");
console.log(inventario.size);            // 2

// Las claves pueden ser de cualquier tipo
const objClave = { id: 1 };
inventario.set(objClave, "valor especial");
console.log(inventario.get(objClave)); // "valor especial"
```

### 1.6 Iteraciones especiales

#### `for...of`

Recorre directamente los **valores** de cualquier iterable (arrays, strings, Sets, Maps, etc.).

```js
const frutas = ["manzana", "pera", "uva"];
for (const fruta of frutas) {
  console.log(fruta); // "manzana", "pera", "uva"
}

// También funciona con Sets y Maps
const miSet = new Set([10, 20, 30]);
for (const valor of miSet) {
  console.log(valor);
}
```

#### `for...in`

Recorre las **claves (nombres de propiedades)** de un objeto. No se recomienda para arrays.

```js
const auto = { marca: "Toyota", año: 2022, color: "blanco" };
for (const clave in auto) {
  console.log(`${clave}: ${auto[clave]}`);
}
// marca: Toyota
// año: 2022
// color: blanco
```

#### `forEach`

Método propio de los arrays. Ejecuta una función por cada elemento. **No devuelve un nuevo array** y no puede interrumpirse con `break`.

```js
const precios = [100, 200, 300];
precios.forEach((precio, indice) => {
  console.log(`Producto ${indice + 1}: $${precio}`);
});
// Producto 1: $100
// Producto 2: $200
// Producto 3: $300
```

| | `for...of` | `for...in` | `forEach` |
|---|---|---|---|
| Recorre | Valores de iterables | Claves de objetos | Elementos de arrays |
| Usa `break` | ✅ Sí | ✅ Sí | ❌ No |
| Devuelve valor | ❌ No | ❌ No | ❌ No |

## 2. Hoisting, Closures, Callbacks, Scope y `this`

### 2.1 Hoisting

El **hoisting** (elevación) es el comportamiento de JavaScript por el cual las **declaraciones** de variables y funciones se "mueven" al inicio de su contexto de ejecución durante la fase de compilación, antes de que el código se ejecute. Solo se eleva la **declaración**, no la asignación.

#### Hoisting con `var`

```js
console.log(nombre); // undefined (no lanza error porque fue "elevada")
var nombre = "Carlos";
console.log(nombre); // "Carlos"

// Lo que JS realmente ejecuta internamente:
// var nombre;          <- elevada al inicio
// console.log(nombre); <- undefined
// nombre = "Carlos";
// console.log(nombre); <- "Carlos"
```

#### Hoisting con `let` y `const`

Las variables `let` y `const` también son elevadas, pero **no se inicializan**, quedando en la llamada **Zona Muerta Temporal (TDZ — Temporal Dead Zone)** hasta que se alcanza su declaración. Acceder a ellas antes lanza un `ReferenceError`.

```js
console.log(edad); // ❌ ReferenceError: Cannot access 'edad' before initialization
let edad = 25;
```

#### Hoisting de funciones declaradas

Las funciones declaradas con `function` se elevan **completamente** (declaración + cuerpo), por lo que pueden llamarse antes de definirse.

```js
saludar(); // ✅ "Hola!" — funciona por hoisting

function saludar() {
  console.log("Hola!");
}

// ⚠️ Las funciones expresadas NO se elevan
despedirse(); // ❌ TypeError: despedirse is not a function
var despedirse = function() {
  console.log("Adiós!");
};
```

### 2.2 Scope (Ámbito)

El **scope** define desde dónde se puede acceder a una variable, función u objeto en el código.

#### Scope Global

Variables declaradas fuera de cualquier función o bloque. Son accesibles desde cualquier parte del programa.

```js
const appNombre = "MiApp"; // scope global

function mostrar() {
  console.log(appNombre); // ✅ accesible aquí
}
mostrar();
```

#### Scope de Función

Variables declaradas dentro de una función. Solo existen dentro de ella.

```js
function calcular() {
  const resultado = 42; // solo existe aquí
  console.log(resultado); // ✅
}
calcular();
console.log(resultado); // ❌ ReferenceError
```

#### Scope de Bloque

Variables declaradas con `let` o `const` dentro de un bloque `{}`. Solo son accesibles dentro de ese bloque.

```js
if (true) {
  let mensaje = "Dentro del bloque";
  console.log(mensaje); // ✅
}
console.log(mensaje); // ❌ ReferenceError

// var IGNORA el scope de bloque:
if (true) {
  var global = "Soy global";
}
console.log(global); // ✅ "Soy global" — porque var no respeta bloques
```

#### Ámbito Léxico

Una función puede acceder a las variables del entorno donde fue **definida**, no donde fue llamada.

```js
const saludo = "Hola";

function externa() {
  const nombre = "Ana";

  function interna() {
    console.log(saludo + ", " + nombre); // accede al scope de externa y al global
  }

  interna();
}
externa(); // "Hola, Ana"
```

#### Variables sin declaración explícita (mala práctica)

Asignar un valor a una variable sin `var`, `let` o `const` la convierte automáticamente en **global**, lo que puede causar bugs difíciles de rastrear.

```js
function crear() {
  sinDeclarar = "Soy global accidentalmente"; // ⚠️ mala práctica
}
crear();
console.log(sinDeclarar); // "Soy global accidentalmente"
```

### 2.3 Closures

Un **closure** es una función que **recuerda el entorno léxico donde fue creada**, manteniendo acceso a las variables de su función externa incluso después de que esta haya terminado de ejecutarse.

```js
function crearContador() {
  let conteo = 0; // variable local de la función externa

  return function() {        // función interna — forma el closure
    conteo++;
    console.log(conteo);
  };
}

const contador = crearContador();
contador(); // 1
contador(); // 2
contador(); // 3
// La variable `conteo` persiste gracias al closure
```

Uso práctico — datos privados:

```js
function crearCuenta(saldoInicial) {
  let saldo = saldoInicial; // no accesible desde fuera

  return {
    depositar(monto) { saldo += monto; },
    retirar(monto)  { saldo -= monto; },
    verSaldo()      { return saldo; }
  };
}

const cuenta = crearCuenta(1000);
cuenta.depositar(500);
cuenta.retirar(200);
console.log(cuenta.verSaldo()); // 1300
console.log(cuenta.saldo);      // undefined — no es accesible directamente
```

### 2.4 Bucle de Eventos (Event Loop)

JavaScript es **monohilo**: solo puede ejecutar una tarea a la vez. Para manejar operaciones asincrónicas (como peticiones HTTP, temporizadores, eventos) sin bloquear el programa, usa el **bucle de eventos**.

**Componentes:**

- **Call Stack:** donde se ejecutan las funciones sincrónicas, una a la vez.
- **Web APIs / Node APIs:** donde viven los temporizadores, eventos, fetch, etc.
- **Cola de tareas (Task Queue):** donde esperan los callbacks listos para ejecutarse.
- **Event Loop:** revisa constantemente si el Call Stack está vacío; si es así, mueve el siguiente callback de la cola.

```js
console.log("1 - Inicio");

setTimeout(() => {
  console.log("3 - Después del timeout"); // va a la cola de tareas
}, 0);

console.log("2 - Fin");

// Salida:
// 1 - Inicio
// 2 - Fin
// 3 - Después del timeout   ← aunque el delay sea 0, espera al stack vacío
```

### 2.5 Callbacks

Un **callback** es una función que se pasa como argumento a otra función y se ejecuta cuando una tarea (normalmente asincrónica) termina.

```js
function descargarDatos(url, alTerminar) {
  console.log("Descargando de:", url);
  // Simulamos una operación asincrónica
  setTimeout(() => {
    const datos = { usuario: "Ana", rol: "admin" };
    alTerminar(datos); // ejecutamos el callback con los datos
  }, 1000);
}

descargarDatos("https://api.ejemplo.com/usuario", (datos) => {
  console.log("Datos recibidos:", datos);
});
```

#### Callback Hell

Anidar muchos callbacks hace el código difícil de leer y mantener:

```js
// ❌ Callback hell
login(usuario, () => {
  obtenerPerfil(usuario, () => {
    cargarFotos(perfil, () => {
      mostrarUI(() => {
        // ...cada vez más difícil de leer
      });
    });
  });
});

// ✅ Solución: usar Promesas o Async/Await (ver sección 4)
```

### 2.6 `this` en JavaScript

El valor de `this` **cambia según el contexto** en el que se ejecuta una función.

#### `this` en el scope global

- Sin `"use strict"`: `this` apunta al objeto global (`window` en navegadores, `global` en Node).
- Con `"use strict"`: `this` es `undefined`.

```js
console.log(this); // window (en el navegador)

"use strict";
function mostrar() {
  console.log(this); // undefined
}
mostrar();
```

#### `this` en métodos de objetos

Dentro de un método, `this` apunta al **objeto que contiene el método**.

```js
const perro = {
  nombre: "Rex",
  ladrar() {
    console.log(this.nombre + " dice: ¡Guau!");
  }
};
perro.ladrar(); // "Rex dice: ¡Guau!"
```

#### `this` en funciones flecha

Las funciones flecha **no tienen su propio `this`**. Heredan el `this` del contexto donde fueron definidas (ámbito léxico).

```js
const reloj = {
  marca: "Casio",
  iniciar() {
    setTimeout(() => {
      console.log(this.marca); // ✅ "Casio" — hereda el this de iniciar()
    }, 1000);
  },
  iniciarNormal() {
    setTimeout(function() {
      console.log(this.marca); // ❌ undefined — this es window/global aquí
    }, 1000);
  }
};

reloj.iniciar();
reloj.iniciarNormal();
```

## 3. Promesas, Async/Await, Prototipos, Clases y Modularidad

### 3.1 Promesas

Una **promesa** es un objeto que representa el resultado eventual (o el fracaso) de una operación asincrónica. Evita el callback hell y hace el código más legible.

**Estados posibles:**

| Estado | Descripción |
|---|---|
| `pending` | La operación aún no ha terminado |
| `fulfilled` | La operación se completó con éxito |
| `rejected` | Ocurrió un error durante la operación |

#### Crear una promesa

```js
const promesa = new Promise((resolve, reject) => {
  const exito = true;

  if (exito) {
    resolve("Datos obtenidos correctamente"); // cumplida
  } else {
    reject("Error al obtener los datos");     // rechazada
  }
});
```

#### Manejar una promesa con `.then()` y `.catch()`

```js
promesa
  .then((resultado) => {
    console.log("✅ Éxito:", resultado);
  })
  .catch((error) => {
    console.log("❌ Error:", error);
  })
  .finally(() => {
    console.log("Operación finalizada"); // siempre se ejecuta
  });
```

#### Encadenar promesas

```js
fetch("https://jsonplaceholder.typicode.com/users/1")
  .then(respuesta => respuesta.json())      // primera promesa
  .then(usuario => {
    console.log("Usuario:", usuario.name);  // segunda promesa
    return usuario.id;
  })
  .catch(error => console.error(error));
```

---

### 3.2 Async / Await

`async/await` es una sintaxis que simplifica el trabajo con promesas, haciendo que el código asincrónico se lea como si fuera sincrónico.

#### Función `async`

Declarar una función como `async` hace que **siempre devuelva una promesa**.

```js
async function obtenerUsuario() {
  return "Ana"; // equivale a: return Promise.resolve("Ana")
}

obtenerUsuario().then(console.log); // "Ana"
```

#### Operador `await`

Pausa la ejecución de la función `async` hasta que la promesa se resuelva. Solo puede usarse dentro de funciones `async`.

```js
async function obtenerDatos() {
  const respuesta = await fetch("https://jsonplaceholder.typicode.com/users/1");
  const usuario   = await respuesta.json();
  console.log("Nombre:", usuario.name);
}

obtenerDatos();
```

#### Manejo de errores con `try/catch`

```js
async function cargarUsuario(id) {
  try {
    const respuesta = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);

    if (!respuesta.ok) throw new Error("Usuario no encontrado");

    const usuario = await respuesta.json();
    console.log("✅", usuario.name);
  } catch (error) {
    console.error("❌ Error:", error.message);
  }
}

cargarUsuario(1);   // ✅ Leanne Graham
cargarUsuario(999); // ❌ Error: Usuario no encontrado
```

### 3.3 Prototipos

En JavaScript, los objetos heredan propiedades y métodos de otros objetos a través de la **herencia prototípica**. Cada objeto tiene una propiedad interna `[[Prototype]]` (accesible como `__proto__`) que apunta a otro objeto.

#### Funciones constructoras y prototipos

```js
// Función constructora
function Persona(nombre, edad) {
  this.nombre = nombre;
  this.edad   = edad;
}

// Los métodos se añaden al prototipo para ser compartidos por todas las instancias
Persona.prototype.saludar = function() {
  console.log(`Hola, soy ${this.nombre} y tengo ${this.edad} años.`);
};

const ana  = new Persona("Ana", 28);
const luis = new Persona("Luis", 35);

ana.saludar();  // "Hola, soy Ana y tengo 28 años."
luis.saludar(); // "Hola, soy Luis y tengo 35 años."

// El método NO está duplicado en cada objeto — vive en el prototipo
console.log(ana.saludar === luis.saludar); // true
```

#### Cadena de prototipos

```js
console.log(ana.__proto__ === Persona.prototype); // true
console.log(Persona.prototype.__proto__ === Object.prototype); // true
```

### 3.4 Clases

Las **clases** son una forma más clara y moderna de crear objetos con propiedades y métodos compartidos. Internamente siguen usando prototipos.

#### Declaración básica

```js
class Animal {
  constructor(nombre, sonido) {
    this.nombre  = nombre;
    this.sonido  = sonido;
  }

  hacerSonido() {
    console.log(`${this.nombre} hace: ${this.sonido}`);
  }
}

const gato = new Animal("Gato", "Miau");
gato.hacerSonido(); // "Gato hace: Miau"
```

#### Herencia con `extends` y `super`

`extends` permite que una clase herede de otra. `super()` llama al constructor de la clase padre.

```js
class Perro extends Animal {
  constructor(nombre, raza) {
    super(nombre, "Guau"); // llama al constructor de Animal
    this.raza = raza;
  }

  presentarse() {
    console.log(`Soy ${this.nombre}, raza ${this.raza}.`);
  }
}

const rex = new Perro("Rex", "Labrador");
rex.hacerSonido(); // "Rex hace: Guau" (heredado)
rex.presentarse(); // "Soy Rex, raza Labrador."
```

#### Métodos estáticos

Los métodos estáticos pertenecen a la clase, no a las instancias.

```js
class Matematica {
  static sumar(a, b) {
    return a + b;
  }
}

console.log(Matematica.sumar(3, 4)); // 7
// new Matematica().sumar(3,4) ❌ los métodos estáticos no se usan en instancias
```

### 3.5 Modularidad

A medida que el código crece, la **modularidad** permite dividirlo en archivos independientes (módulos), cada uno con una responsabilidad clara. Esto mejora el mantenimiento, la reutilización y la escalabilidad.

> Los módulos ES usan la extensión `.mjs` o requieren `type="module"` en el `<script>` HTML, o `"type": "module"` en `package.json`.

#### Exportación nombrada

Permite exportar múltiples elementos desde un módulo.

```js
// archivo: utils.js
export const PI = 3.14159;

export function areaCirculo(radio) {
  return PI * radio * radio;
}

export function perimetroCirculo(radio) {
  return 2 * PI * radio;
}
```

```js
// archivo: main.js
import { PI, areaCirculo, perimetroCirculo } from "./utils.js";

console.log(PI);                   // 3.14159
console.log(areaCirculo(5));       // 78.53975
console.log(perimetroCirculo(5));  // 31.4159
```

#### Exportación por defecto

Solo puede haber **una exportación por defecto** por módulo. Se usa para exportar el elemento principal.

```js
// archivo: saludo.js
export default function saludar(nombre) {
  return `¡Hola, ${nombre}!`;
}
```

```js
// archivo: main.js
import saludar from "./saludo.js"; // se puede nombrar diferente al importar

console.log(saludar("María")); // "¡Hola, María!"
```

#### Combinar exportaciones

```js
// archivo: app.js
import saludar from "./saludo.js";
import { areaCirculo } from "./utils.js";

console.log(saludar("Dev"));
console.log(areaCirculo(3));
```

#### Importación dinámica

Carga un módulo **de forma asincrónica** solo cuando es necesario, mejorando el rendimiento en aplicaciones grandes.

```js
async function cargarModulo() {
  const { areaCirculo } = await import("./utils.js");
  console.log(areaCirculo(4)); // 50.26...
}

// Útil para cargar funciones pesadas solo cuando el usuario las necesita
document.getElementById("calcular").addEventListener("click", cargarModulo);
```

#### Ventajas de la modularidad

| Ventaja | Descripción |
|---|---|
| **Reutilización** | Un módulo puede importarse en múltiples partes del proyecto |
| **Mantenimiento** | Código organizado en archivos pequeños y específicos |
| **Escalabilidad** | Facilita agregar nuevas funcionalidades sin romper lo existente |
| **Encapsulamiento** | Las variables internas del módulo no contaminan el scope global |

## Resumen General

| Concepto | Qué es |
|---|---|
| **Hoisting** | Las declaraciones de variables y funciones se elevan al inicio del contexto |
| **Scope** | Controla la visibilidad de variables según dónde están declaradas |
| **Closures** | Funciones que recuerdan el entorno donde fueron creadas |
| **Event Loop** | Mecanismo que permite manejar tareas asincrónicas sin bloquear el hilo |
| **Callbacks** | Funciones pasadas como argumento para ejecutarse al completar una tarea |
| **`this`** | Referencia al objeto que depende del contexto de ejecución |
| **Promesas** | Representan el resultado futuro de una operación asincrónica |
| **Async/Await** | Sintaxis que simplifica el trabajo con promesas |
| **Prototipos** | Mecanismo de herencia entre objetos en JS |
| **Clases** | Sintaxis moderna y clara para crear objetos con herencia |
| **Módulos** | Archivos que exportan e importan código para organizar el proyecto |
